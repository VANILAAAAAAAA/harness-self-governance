# v2.0 Session Knowledge

## Purpose

Session compression turns optional local raw transcripts into compact external knowledge for the v2.0 governance graph and dashboard. It is deterministic and does not call external APIs.

## Inputs

Default command:

```bash
python -m graph_harness_maintain sessions compress --input sessions/raw
```

Accepted input formats:

- `.md`
- `.txt`
- `.json`

If the input directory does not exist, compression writes an empty local session index with a warning and exits successfully.

## Outputs

| Path | Purpose | Privacy |
| --- | --- | --- |
| `artifacts/v2/sessions/session-index.json` | index of compressed summaries | `local_only` |
| `artifacts/v2/sessions/<session_id>.summary.json` | one deterministic summary per source | `local_only` |

Raw sessions remain under `sessions/raw/` or `sessions/private/` and are ignored by git.

## Summary schema

```json
{
  "schema_version": "2.0",
  "session_id": "...",
  "title": "...",
  "source_hash": "sha256:...",
  "privacy": "local_only",
  "summary": "...",
  "decisions": [],
  "requirements": [],
  "constraints": [],
  "commands": [],
  "artifacts_referenced": [],
  "open_questions": [],
  "graph_links": []
}
```

## Extraction behavior

The compressor extracts likely decisions, requirements, constraints, commands, artifact paths, branch names, PR numbers, tags, and open questions with local regex/prefix rules. `source_hash` is computed from source bytes.

## Redaction

Obvious token-like values such as API keys, GitHub tokens, and long structured bearer-style tokens are replaced with `[REDACTED_TOKEN]`. Session summaries are marked `local_only` and are not a sensitive export channel.

## Safety boundary

- Do not commit raw sessions.
- Do not export secrets.
- Do not require external API calls.
- Do not mutate source transcripts.
- Do not perform destructive apply, graph mutation execution, or sensitive export.
