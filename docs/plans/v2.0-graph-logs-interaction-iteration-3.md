# v2.0 Graph/Logs Interaction Iteration 3 Plan

> **Mode:** read-only UI foundation. No destructive operations, no graph/events mutation, no raw session export.

## Feedback mapping

| User feedback | Implementation response |
| --- | --- |
| Graph is good but dense; edge should carry more work | Add first-class edge filters, visible edge type chips, edge-specific styling, edge click highlighting, confidence/relation/source/target inspector fields. |
| Dense graph should remain usable | Add traversal-hub focus, graph diagnostic summary, degree hubs, component/density/isolated/broken-edge metrics. |
| Logs need true interactive up/down scrolling | Add explicit `↑ table` / `↓ table` and `↑ preview` / `↓ preview` controls backed by `scrollBy`, with independent scroll containers; after follow-up feedback, make the whole Logs route vertically scrollable at 100% browser height and add an explicit sticky scroll hint. |
| Logs need stronger human readability | Add line-numbered previews, clearer table path column, sticky table header, preview scroll toolbar, and lineage retained below preview. |
| Graph edge click needs to actually open inspector | Add wider transparent SVG edge hit targets, keep visible edge/label pointer events active, and prevent canvas pan from swallowing edge clicks. |
| Graph `View in Logs` should locate the backing file | Track selected node/edge log path, route to Logs, select matching inventory row, scroll it into view, and update preview. |
| Future plan: graph summary and AI summary/diagnosis | Add deterministic local `graph_summary` as the substrate for future AI diagnosis: density, components, broken endpoints, hubs, isolated nodes, type counts. |

## Acceptance criteria

- Graph page:
  - `DATA.edge_filter_types` exists and renders edge chips.
  - Edge filter click changes visible edge count without mutating graph data.
  - Edge click highlights the selected SVG path and updates inspector metadata.
  - Summary inspector shows density, components, isolated nodes, broken edges, diagnostics, and traversal hubs.
  - Focus hubs recenters a high-degree node.
- Logs page:
  - Whole Logs route scrolls vertically at 100% browser height when the layout is taller than the viewport.
  - Sticky `Scroll this Logs page` hint is visible above the three-pane layout.
  - Table and preview have explicit up/down scroll buttons.
  - Table and preview scroll containers have `scrollHeight > clientHeight` when content is long.
  - Preview text is line-numbered for human readability.
  - File table and preview scroll independently.
- Safety:
  - Static dashboard only; no external CDN, no backend, no destructive action buttons.
  - Screenshots are generated only for QA and removed before commit.
