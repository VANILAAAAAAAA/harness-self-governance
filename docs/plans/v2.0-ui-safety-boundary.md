# v2.0 UI Safety Boundary

## Boundary statement

The v2.0 dashboard is a read-only local visualization. It displays governance graph data, pipeline status, blockers, warnings, artifacts, gates, proposal status, provenance, compressed sessions, tools, knowledge, and safety state. It does not provide or execute any mutation pathway.

## Dashboard implementation constraints

| Constraint | Required behavior |
| --- | --- |
| Static file | `artifacts/v2/dashboard/index.html` |
| CSS | embedded |
| Data | embedded JSON or linked local JSON only |
| External dependencies | none; no CDN, npm, or server |
| Browser mode | works from local filesystem |
| Operations | read-only display only |

## Explicitly blocked operations

- destructive apply
- delete
- move
- quarantine
- rehydrate
- raw archive apply execution
- graph mutation execution
- graph/events mutation
- sensitive export
- push, tag, release, PyPI publish, or other remote publication

## Raw session boundary

Raw transcripts are local-only and must remain ignored by git:

- `sessions/raw/`
- `sessions/private/`

Compressed summaries under `artifacts/v2/sessions/` are also generated local artifacts and remain ignored.

## Artifact boundary

All v2 generated files live under `artifacts/v2/`, which is ignored. Documentation may describe schemas and commands, but generated graph/dashboard/session/pipeline outputs are not committed.

## Provenance and proposal boundary

The dashboard may show proposal and provenance status. It must not add approval buttons, apply buttons, graph mutation controls, destructive action controls, raw archive apply controls, or sensitive export controls.

## Verification

Run:

```bash
python -m graph_harness_maintain dashboard build
python -m graph_harness_maintain pipeline v2.0-rc
```

Expected safety fields in `artifacts/v2/pipeline-run.json`:

```json
{
  "read_only_ui": true,
  "proposal_only": true,
  "destructive_operations_allowed": false,
  "graph_mutation_allowed": false,
  "remote_publication_allowed": false,
  "sensitive_export_allowed": false,
  "session_raw_committed": false
}
```
