# v2.0 Graph Dashboard Roadmap

## Scope

v2.0 establishes a lightweight, read-only visualization/dashboard foundation for the governance harness. It renders the governance graph, logic flow, tools, knowledge structure, artifacts, gates, proposals, provenance, release audits, and compressed session knowledge as local artifacts.

## Read-only boundary

The dashboard and graph export do not execute apply behavior. They do not delete, move, quarantine, rehydrate, apply raw archives, mutate graph/event stores, publish remotely, or export sensitive data.

## Artifact layout

Generated outputs remain local under ignored paths:

| Path | Purpose | Git policy |
| --- | --- | --- |
| `artifacts/v2/graph/governance-graph.json` | deterministic governance graph projection | ignored |
| `artifacts/v2/dashboard/index.html` | single-file static dashboard | ignored |
| `artifacts/v2/sessions/session-index.json` | local-only compressed session index | ignored |
| `artifacts/v2/sessions/*.summary.json` | local-only session summaries | ignored |
| `artifacts/v2/pipeline-run.json` | v2 local pipeline report | ignored |
| `sessions/raw/` | raw local transcripts | ignored |
| `sessions/private/` | private local session material | ignored |

## Dashboard layout

1. System Health status cards
2. Governance Graph node/edge inventory
3. Logic Flow from v1 checks through v2 visualization outputs
4. Tools and Knowledge matrix
5. Sessions panel for compressed local-only knowledge
6. Artifacts panel for generated v2 outputs
7. Safety Boundary panel for blocked operations
8. Warnings and Blockers panel

## Milestones

1. Add deterministic graph export.
2. Add local-only session compression without external API calls.
3. Add single-file static dashboard with embedded CSS and local JSON data.
4. Integrate `pipeline v2.0-rc`.
5. Validate CLI help, tests, safety gates, and ignored artifact paths.

## Non-goals

- No destructive apply.
- No graph mutation execution.
- No raw archive apply execution.
- No delete/move/quarantine/rehydrate execution.
- No sensitive export.
- No external dashboard service, CDN, npm, or server requirement.
