# v2.0 Agent-Triggered Archive Contract

v2.0 keeps the Governance Hub static, read-only, local-first, and limited to the Graph and Logs pages. The Hub does not call an LLM, does not expose local model/provider settings, and does not require a backend server. Session and project compilation is performed by an agent outside the Hub and then written as structured local artifacts that the dashboard reads.

## User flow

A user can tell Hermes:

```text
Archive this session into profile general, project harness-self-governance.
```

The agent may summarize the current conversation/project context, but the dashboard itself never invokes a Hub-side LLM API.

## Agent-written artifacts

For profile `general` and project `harness-self-governance`, the agent creates or updates:

```text
artifacts/v2/projects/general/harness-self-governance/project-summary.json
artifacts/v2/projects/general/harness-self-governance/decision-ledger.json
artifacts/v2/projects/general/harness-self-governance/requirements.json
artifacts/v2/projects/general/harness-self-governance/constraints.json
artifacts/v2/projects/general/harness-self-governance/session-index.json
```

All of these artifacts are local generated v2 artifacts. They remain ignored by git with the rest of `artifacts/v2/`.

## Validation flow

After writing the archive artifacts, the agent runs:

```bash
python -m graph_harness_maintain project validate --profile general --project harness-self-governance
python -m graph_harness_maintain graph export
python -m graph_harness_maintain lineage build
python -m graph_harness_maintain lineage validate
python -m graph_harness_maintain dashboard build
python -m graph_harness_maintain pipeline v2.0-rc
```

## Safety boundary

- No Hub-side LLM API.
- No local server model/provider API.
- No model/provider settings UI.
- Dashboard is static and read-only.
- Raw sessions remain local-only and ignored.
- Generated v2 artifacts remain local and ignored.
- No destructive apply, graph mutation execution, raw archive apply execution, or sensitive export.
- `general` remains the governance hub; `ehrlab` is represented only by sanitized profile/project metadata until an agent writes local structured artifacts.
