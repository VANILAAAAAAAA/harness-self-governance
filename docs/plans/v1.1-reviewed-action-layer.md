# v1.1 Reviewed Action Layer Implementation Plan

> **For Hermes:** Implement locally only. Do not commit, push, tag, release, publish, force-push, delete, move, quarantine, rehydrate, or perform destructive operations.

**Goal:** Build and locally test a v1.1 reviewed-action layer on top of the v1.0 read-only governance pipeline.

**Architecture:** Keep v1.0 read-only pipeline intact. Add v1.1-only commands that generate local artifacts under `artifacts/v1.1/`, validate proposal/template structure, report adapter maintenance status, and append local-test provenance events without mutating protected graph/event stores. Execution remains review-gated and proposal-only; no apply execution command is added.

**Tech Stack:** Python 3.11 stdlib, argparse CLI, pytest, existing `graph_harness_maintain` package.

## Safety constraints

- No commit unless explicitly approved later.
- No push, tag, GitHub Release, PyPI publish, force push, destructive file operations, quarantine, rehydrate, or provenance upgrade.
- Before any future commit attempt verify `git var GIT_AUTHOR_IDENT` and `git var GIT_COMMITTER_IDENT`; both must be the required user-owned identity.
- Allowed local mutations: source/test/docs edits and generated local artifacts under `artifacts/v1.1/`.
