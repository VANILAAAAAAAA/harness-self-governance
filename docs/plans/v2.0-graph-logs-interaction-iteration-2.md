# v2.0 Graph/Logs Interaction Iteration 2 Plan

> **Mode:** build UI foundation only. Keep the dashboard read-only, local-only, no generated screenshots committed.

## Feedback mapping

| User feedback | Implementation response |
| --- | --- |
| Graph should support left/right mouse drag | Add viewport panning for left-button and right-button drag on the SVG canvas; keep node drag on nodes; suppress context menu on graph canvas. |
| Logs still cannot scroll | Rework Logs middle pane to use a fixed-height CSS grid/flex scroll area: `min-height: 0`, `overflow-y: auto`, sticky headers, and body-level overflow containment. |
| Logs left side should highlight pointer target | Add hover/highlight cursor styles for folder rows and file rows; active folder remains visually distinct. |
| Continue next iteration planning | Record future v2 UI work: graph clustering, edge filters, settings/profile/admin, richer lineage drilldown. |

## Acceptance criteria

- `frontend-visual-qa-capture --serve-dir artifacts/v2/dashboard --out artifacts/v2/screenshots` produces:
  - `artifacts/v2/screenshots/graph-page.png`
  - `artifacts/v2/screenshots/logs-page.png`
- Graph page:
  - canvas has pan state and left/right mouse drag handlers;
  - right-click context menu is disabled only inside graph canvas;
  - node drag still works independently;
  - filters/search/edge metadata inspector remain functional.
- Logs page:
  - left explorer folder rows are clickable and visibly hoverable;
  - file rows are hoverable/clickable;
  - middle file table scrolls independently and does not hide lower rows;
  - preview panel remains independently scrollable.
- Generated screenshots are deleted before commit.

## Implementation tasks

1. **Tests**
   - Extend `tests/test_dashboard.py` to assert presence of:
     - `startCanvasPan`
     - `finishCanvasPan`
     - `contextmenu`
     - `buttons === 1 || buttons === 2`
     - `.folder-row:hover`
     - `tr:hover`
     - `.file-table-scroll`
     - `min-height:0`

2. **Graph canvas drag**
   - Modify `src/graph_harness_maintain/dashboard.py`.
   - Add canvas-level `mousedown`, `mousemove`, `mouseup`, and `mouseleave` handlers.
   - For canvas background/edge area only, pan `panZoomState.x/y` with left or right mouse drag.
   - Keep node drag guarded by `stopPropagation()`.

3. **Logs scrolling and hover**
   - Make `.logs-layout`, `.file-table`, `.file-table-scroll`, `.preview-panel`, and `.explorer` obey `min-height:0` and independent overflow.
   - Add pointer hover styles for `.folder-row`, tree file items, and file table rows.

4. **Visual QA loop**
   - Build dashboard.
   - Run capture command.
   - Inspect screenshots against concept images.
   - Apply at least one layout fix if screenshots reveal a clear mismatch.
   - Delete screenshots with `rm -rf artifacts/v2/screenshots` before commit.

5. **Validation and commit**
   - Run pytest, v2 pipeline, CLI smoke, hygiene scan.
   - Confirm generated artifacts/screenshots/raw sessions/concept images are untracked or ignored and not staged.
   - Commit only docs/source/tests.
