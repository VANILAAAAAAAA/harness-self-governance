# v2.0 Graph Schema

## Purpose

The v2.0 graph export is a deterministic, read-only projection of the local governance state. It supports visualization and auditability only; it does not mutate source graph/events stores or execute maintenance actions.

## Export path

`artifacts/v2/graph/governance-graph.json`

## Top-level schema

```json
{
  "schema_version": "2.0",
  "generated_at": "ISO-8601 timestamp",
  "summary": {},
  "nodes": [],
  "edges": [],
  "warnings": [],
  "blockers": []
}
```

`nodes` and `edges` are sorted by stable `id`. `generated_at` records export time; other content is deterministic for a fixed repository/artifact state.

## Required node types

| Type | Boundary |
| --- | --- |
| `pipeline_run` | observed local pipeline report |
| `artifact` | generated local file or directory |
| `report` | generated local report |
| `proposal` | proposal-only reviewed action manifest |
| `gate` | approval/safety gate |
| `policy` | local safety policy |
| `adapter` | read-only adapter capability/report node |
| `tool` | local CLI/tool dependency |
| `knowledge_source` | docs, compressed session index, or safe local knowledge |
| `session` | compressed local-only session summary |
| `decision` | architecture/safety decision |
| `requirement` | hard v2.0 requirement |
| `provenance_state` | provenance current-state artifact |
| `test_result` | local validation output |
| `release_audit` | release surface audit output |

## Required edge types

| Type | Meaning |
| --- | --- |
| `generated` | pipeline or action generated artifact/report |
| `validates` | validation relation |
| `blocks` | gate or policy blocks risky operation |
| `references` | safe reference relation |
| `derived_from` | derived projection/input relation |
| `governed_by` | governed by policy/gate |
| `reports_on` | report describes target |
| `decided` | decision establishes requirement |
| `requires` | source requires target |
| `uses_tool` | pipeline/action uses local tool |
| `summarized_into` | session/artifact summarized into index/graph |

## Included concepts

The graph must include v1 local pipeline, v1.1 proposal pipeline, approval gates, adapter report, provenance current-state, release audit, generated artifacts, and session knowledge when available.

## Safety boundary

- No destructive apply.
- No graph mutation execution.
- No sensitive export.
- Raw sessions are local-only and ignored by git.
- The export is a local artifact under `artifacts/v2/`.
