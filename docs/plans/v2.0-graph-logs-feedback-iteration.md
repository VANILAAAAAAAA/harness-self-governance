# v2.0 Graph/Logs Feedback Iteration Plan

> **For Hermes:** Implement this plan with TDD. Keep the dashboard read-only and local-only.

**Goal:** Expand the v2.0 dashboard from visual approximation into a more useful Graph/Logs inspection surface based on user feedback.

**Architecture:** Keep a single static `artifacts/v2/dashboard/index.html` app with embedded JSON data. Expand graph export with safe repository-local capability nodes, add client-side graph filtering/interactions, and improve Logs navigation with clickable folder groups plus scrollable file browsing.

**Tech Stack:** Python stdlib, pytest, vanilla HTML/CSS/JS, local generated JSON artifacts only.

---

## Feedback mapping

| User feedback | v2.0 iteration response |
| --- | --- |
| Graph has too few nodes/edges | Add read-only capability inventory nodes for tools, modules, tests, docs, policies, and local skills where available. |
| Add tools and skills associations | Add tool/skill/module/knowledge relations with deterministic IDs and metadata. |
| Need filters due to node growth | Add client-side type filters, search filtering, selected count, and reset/fit behavior. |
| Edge metadata interaction is good | Preserve edge click inspector and expose relation/source/target metadata. |
| Logs folder cannot click | Make explorer groups clickable and update table content by group. |
| Middle log cannot scroll | Make file table body independently scrollable with sticky header/footer within viewport. |
| Many buttons still static | Prioritize functional graph/log interactions now; add lightweight theme toggle as a safe visual/local interaction. Settings/profile remain v2.1 placeholders. |

## Safety boundary

- Dashboard stays read-only.
- Logs inspect generated/local repository artifacts only.
- No apply, delete, move, quarantine, rehydrate, graph mutation execution, sensitive export, network fetch, npm, CDN, push, tag, release, or publish.
- Raw sessions remain local-only and ignored.

### Task 3: Expand graph export safely

**Files:**
- Modify: `src/graph_harness_maintain/graph_export.py`
